## How to work in this repo (paid-for gotchas)

- **Bun isn't on the default PATH:** `export PATH="$HOME/.bun/bin:$PATH"` in every shell. Then
  `bun install`. Keep green before any commit: `bun test`, `bunx tsc --noEmit`,
  `bunx biome check ./src ./tests` (and `bunx biome check --write` to auto-fix format + import sort).
- **Tests use a fake client** at `tests/helpers/fake-plane-client.ts`. When you add a `PlaneClient`
  method, add it to the fake too, or real-import tests throw (this bit us with `projectBoardUrl`).
- **Edit-tool tab gotcha:** biome indents with TABS. Matching leading whitespace in the Edit tool
  fails silently. Match an inner substring (no leading whitespace) and let `biome --write` reindent.
- **Live testing:** use the **Bloomenkohlberg SANDBOX** project (identifier BLOOM), workspace slug
  `bloomenkohlberg`, creds already in `.env`. NEVER test against the real **Data Platform** board
  (identifier DATA, ~815 production items). Clean up sandbox items with `delete ... --yes`. The
  `planestories` and `archived` labels already exist in BLOOM from prior testing.
- **Secrets:** `.env` holds real credentials, gitignored. Never print or commit them.

## Plane REST facts already discovered (don't rediscover)

- CRUD path segment is `/issues/` (not `/work-items/`). Archive is `/work-items/{id}/archive/` but
  Plane only archives completed/cancelled items — so we archive via a **label** instead (there is no
  `archiveWorkItem` on the client; `delete --archive` applies the `archived` label).
- The `external_id` filter returns **HTTP 404** (not an empty list) on a miss. `client.request` has
  an `allowNotFound` option; `findWorkItemByExternalId` uses it.
- The list endpoint returns `description_html` by default; pass `expand=state,assignees,labels` for
  related names. `fetchWorkItems` already does this.
- 429/rate-limit handling belongs in `client.request` (that's slice 1). `fetchProjectIndex`
  (slice 4 shared infra) should extend `fetchWorkItems` in `src/plane/issues.ts`, not add a parallel
  path.
